# Plan 6.2: FFmpeg Optimization

Optimize FFmpeg encoding parameters to reduce CPU and memory usage while maintaining acceptable quality.

## Overview

FFmpeg encoding is the primary CPU consumer in the streaming pipeline. By adjusting resolution, FPS, and encoding presets, we can significantly reduce resource usage with minimal quality impact.

## Tasks

### 3. Encoding Preset Optimization
- [ ] Implement `ultrafast` preset for real-time encoding
- [ ] Use `baseline` profile for maximum compatibility
- [ ] Set `tune=zerolatency` for streaming
- [ ] Adjust bitrate for SD resolution (500kbps vs 2000kbps)

**FFmpeg Parameters:**
```bash
# Low-resource encoding
ffmpeg -rtsp_transport tcp \
  -i rtsp://camera \
  -vf "scale=640:360" \
  -r 2 \
  -c:v libx264 \
  -preset ultrafast \
  -tune zerolatency \
  -profile:v baseline \
  -level 3.0 \
  -b:v 500k \
  -maxrate 500k \
  -bufsize 1000k \
  -an \
  -f mpegts -
```

### 4. Hardware Acceleration
- [ ] Detect and enable VAAPI if available
- [ ] Enable NVDEC/NVENC for NVIDIA GPUs
- [ ] Fallback to software encoding gracefully
- [ ] Add detection for Quick Sync (Intel)

**Detection Logic:**
```typescript
function getEncoderConfig(): EncoderConfig {
  if (process.env.USE_HARDWARE_ACCEL === 'vaapi') {
    return {
      codec: 'h264_vaapi',
      preset: 'default'
    };
  }
  if (process.env.USE_HARDWARE_ACCEL === 'nvidia') {
    return {
      codec: 'h264_nvenc',
      preset: 'll'
    };
  }
  // Software fallback
  return {
    codec: 'libx264',
    preset: 'ultrafast',
    tune: 'zerolatency'
  };
}
```

## Optimization Impact

| Setting | Current | Optimized | Savings |
|---------|---------|-----------|---------|
| Resolution | 1920x1080 | 640x360 | ~75% CPU |
| FPS | 4 | 2 | ~50% CPU |
| Preset | medium | ultrafast | ~40% CPU |
| Bitrate | 2000k | 500k | ~75% bandwidth |

**Combined Effect:**
- CPU: ~85% reduction per stream
- Memory: ~40% reduction per stream
- Bandwidth: ~75% reduction
