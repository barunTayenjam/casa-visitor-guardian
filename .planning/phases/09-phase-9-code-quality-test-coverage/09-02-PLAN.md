  <action>
    Replace all console.* calls with logger.* calls in the 35 files listed above. This is a mechanical migration following a consistent pattern.

    **Logger API (from server/src/utils/logger.ts):**
    - `logger.info(message: string, source?: string, metadata?: Record<string, unknown>)`
    - `logger.warn(message: string, source?: string, metadata?: Record<string, unknown>)`
    - `logger.error(message: string, source?: string, error?: unknown, metadata?: Record<string, unknown>)`
    - `logger.debug(message: string, source?: string, metadata?: Record<string, unknown>)`

    **Replacement rules:**

    1. Add import at top of each file (if not already present):
       `import { logger } from '../utils/logger.js';` (adjust relative path per file location — e.g., `'../../utils/logger.js'` for files in subdirectories like services/review/, routes/, streams/, etc.)

    2. console.log('message') → logger.info('message', 'SOURCE')
    3. console.log('prefix:', value) → logger.info(`prefix: ${value}`, 'SOURCE')
    4. console.warn('message', error) → logger.warn('message', 'SOURCE')
    5. console.error('message', error) → logger.error('message', 'SOURCE', error)
    6. console.error('message') → logger.error('message', 'SOURCE')
    7. console.debug('message') → logger.debug('message', 'SOURCE')

    **Source tag conventions (derive from file purpose):**
    - Files in `routes/` → use route context (e.g., 'API', 'Highlights', 'Motion', 'Notifications', 'Detection', 'Visitor')
    - Files in `services/` → use service name (e.g., 'NotificationService', 'CacheService', 'BatchWorker', 'CleanupService', 'RetryService', 'CircuitBreaker')
    - Files in `controllers/` → use 'Controller' or specific name (e.g., 'NVIDIA', 'System', 'Settings', 'Stream')
    - Files in `streams/` → use 'STREAM' or 'RTSP'
    - Files in `events/` → use 'EventBus'
    - Files in `detection/` → use 'Detection'
    - Files in `utils/` → use the utility name (e.g., 'Cron', 'DataNormalizer')
    - Files in `middleware/` → use 'RateLimit'
    - Files in `config/` → use 'Config'
    - database.ts → use 'Database'
