# Plan 2.4: Event Filters UI

Build event filters UI — search bar, filter dropdowns, pagination for powerful event browsing.

## Overview

Create a comprehensive user interface for filtering and browsing events, allowing users to:
- Search events using multiple filter criteria
- Navigate through paginated results
- See active filters and clear them easily
- Share filtered URLs via query parameters
- Experience fast, responsive filtering with proper loading states

This builds on top of the API created in Plan 2.3.

### 8. useEventSearch Hook
- Create `frontend/src/hooks/useEventSearch.ts`
- Custom hook for event search logic
- Manages filter state
- Integrates with React Query for caching
- Handles pagination
- Refetches on filter change
- Structure:
  ```typescript
  interface UseEventSearchParams {
    filters: EventSearchFilters;
    enabled?: boolean;
  }

  function useEventSearch(params: UseEventSearchParams) {
    return useQuery({
      queryKey: ['events', params.filters],
      queryFn: () => ApiService.searchEvents(params.filters),
      enabled: params.enabled ?? true,
      keepPreviousData: true // Prevent flicker on pagination
    });
  }
  ```

### 10. API Service Integration
- Add to `frontend/src/services/ApiService.ts`:
  ```typescript
  async searchEvents(filters: EventSearchFilters): Promise<EventSearchResponse> {
    const params = new URLSearchParams();
    if (filters.startDate) params.append('startDate', filters.startDate);
    if (filters.endDate) params.append('endDate', filters.endDate);
    // ... all other filters

    const response = await fetch(`/api/events/search?${params}`);
    if (!response.ok) throw new Error('Search failed');
    return response.json();
  }
  ```
