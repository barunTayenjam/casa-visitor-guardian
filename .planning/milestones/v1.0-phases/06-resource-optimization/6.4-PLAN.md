# Plan 6.4: Docker & Infrastructure Optimization

Configure Docker containers and system-level settings for operation on constrained hardware.

## Overview

Docker containers can consume unlimited resources by default. This plan sets appropriate limits and configurations for 1-core, 2GB RAM systems.

## Tasks

### 1. Docker Memory Limits
- [ ] Set explicit memory limits for all containers
- [ ] Configure memory reservations
- [ ] Enable memory swap limit
- [ ] Set OOM kill policies

**docker-compose.yml additions:**
```yaml
services:
  backend:
    mem_limit: 512m
    mem_reservation: 256m
    memsw_limit: 768m
    oom_kill_disable: false
    deploy:
      resources:
        limits:
          memory: 512M
        reservations:
          memory: 256M

  opencv:
    mem_limit: 384m
    mem_reservation: 192m
    memsw_limit: 512m
    deploy:
      resources:
        limits:
          memory: 384M

  frontend:
    mem_limit: 256m
    mem_reservation: 128m
    deploy:
      resources:
        limits:
          memory: 256M

  postgres:
    mem_limit: 384m
    mem_reservation: 192m
    environment:
      POSTGRES_MAX_CONNECTIONS: "20"
      POSTGRES_SHARED_BUFFERS: "64MB"
    deploy:
      resources:
        limits:
          memory: 384M

  redis:
    mem_limit: 64m
    mem_reservation: 32m
    command: redis-server --maxmemory 50mb --maxmemory-policy allkeys-lru
    deploy:
      resources:
        limits:
          memory: 64M
```

### 5. SWAP Configuration
- [ ] Document SWAP requirements
- [ ] Recommend SWAP size (2GB minimum)
- [ ] Set swappiness for container workloads
- [ ] Create setup script

**SWAP Setup (systemd):**
```bash
# Create 2GB swap file
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# Add to fstab
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# Set swappiness (lower = less swapping)
echo 10 | sudo tee /proc/sys/vm/swappiness
echo "vm.swappiness=10" | sudo tee -a /etc/sysctl.conf
```
