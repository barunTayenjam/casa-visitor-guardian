# Plan 5.1: Batch Detection Review & Fixes

Review and fix Batch Detection page to improve API integration, error handling, and overall user experience.

## Overview

The Batch Detection page allows users to run AI detection on historical images. This plan addresses issues with API consistency, error handling, progress tracking, and UX improvements.

## Tasks

### 1. API Integration Audit
- Review all API calls in BatchDetectionPage.tsx
- Ensure consistent use of apiService instead of direct fetch
- Add proper TypeScript types for all responses
- Implement request timeout handling (30 second default)

### 2. Error Handling Improvements
- Add retry logic for transient failures (3 attempts with exponential backoff)
- Improve error messages with specific failure reasons
- Add recovery actions where possible
- Implement connection loss detection and reconnection

### 3. Progress & Loading States
- Add skeleton loaders during initial load
- Improve progress bar accuracy and smoothness
- Add estimated time remaining calculation
- Show processing rate (images/second)

### 4. Job Management
- Add job cancellation functionality
- Implement pause/resume for long jobs
- Add job history with pagination
- Store recent jobs in localStorage for quick access

### 5. Performance Optimization
- Add virtualization for results list (react-window or similar)
- Lazy load images with intersection observer
- Limit initial results display, load more on scroll
- Debounce date selection API calls

### 6. UX Improvements
- Add keyboard shortcuts (Enter to start, Escape to cancel)
- Add confirmation dialog before starting large batch
- Improve mobile responsiveness
- Add toast notifications for status changes
