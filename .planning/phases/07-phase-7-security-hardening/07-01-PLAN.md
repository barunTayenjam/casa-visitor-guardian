<objective>
Fix critical authentication bypass (JWT refresh fallback to jwt.decode()), eliminate hardcoded default seed password 'changeme', and apply Helmet security headers that are installed but never used.

Purpose: These are the three highest-severity security vulnerabilities — an attacker with an expired/tampered JWT can obtain a fresh valid token, production deployments may run with default credentials, and all HTTP responses lack standard security headers.

Output: AuthController with verify-only refresh, production-hardened seed password flow, Helmet-enabled Express app.
</objective>
