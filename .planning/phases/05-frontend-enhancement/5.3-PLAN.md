# Plan 5.3: Events Page Review & Fixes

Review and fix Events page to improve filtering, search, event management, and overall user experience.

## Overview

The Events page displays security events with filtering and search capabilities. This plan addresses improvements to search, filtering, bulk operations, and event management.

## Tasks

### 1. Advanced Filtering
- Add date range picker with preset options (Today, Yesterday, Last 7 days, Last 30 days, Custom)
- Add time-of-day filter (Morning, Afternoon, Evening, Night)
- Add confidence threshold slider (0-100%)
- Add camera multi-select dropdown
- Add event type checkboxes (motion, face, object)
- Add "reviewed/unreviewed" filter toggle
- Persist filter preferences in URL params and localStorage

### 2. Search Improvements
- Add debounced search input (300ms delay)
- Search by: filename, camera name, detected objects
- Add search suggestions/autocomplete
- Highlight matching terms in results
- Show result count in real-time

### 3. Bulk Actions
- Add checkbox selection for multiple events
- Bulk delete with confirmation
- Bulk mark as reviewed
- Bulk export (CSV, JSON, images ZIP)
- "Select all on page" and "Select all matching filter" options
- Show selection count in toolbar

### 4. Event Cards Optimization
- Implement image lazy loading with blur placeholder
- Add skeleton loader while loading
- Show detection confidence as visual indicator
- Add hover state with quick actions
- Optimize for 1000+ events with virtualization

### 5. Keyboard Navigation
- Arrow Up/Down: Navigate events
- Enter: Open event detail
- Delete: Delete selected event (with confirmation)
- Ctrl+A: Select all visible
- Escape: Close modals, clear selection

### 6. Event Detail Modal
- Show full event metadata (timestamp, camera, detections, confidence)
- Display annotated image with bounding boxes
- Allow adding notes to event
- Link to create visitor from unknown face
- Navigation: Previous/Next event buttons
- Share event link functionality

### 7. Performance & UX
- Infinite scroll with pagination
- Show loading indicator for initial load
- Handle empty states with helpful messages
- Add "No events match filters" state
- Implement URL-based filtering for shareability
