  <action>
    Add a `persistCameras()` async method to the `StreamManager` class in `server/src/streams/rtspManager.ts`.

    **CRITICAL SECURITY: Credential re-encryption.** The config loading pipeline at `config/index.ts:300-306` decrypts RTSP stream paths on load. This means `camera.config.streams[n].path` holds the **DECRYPTED** plaintext RTSP URL. Writing this back to `cameras.json` without re-encryption would **leak plaintext credentials to disk** — a security regression.

    The `persistCameras()` method must:
    1. Build a serializable array from `this.cameras.values()`, mapping each Camera to its config.
    2. For each camera config, re-encrypt stream paths: for each stream in `camera.config.streams`, check if the original config had encrypted paths (use `isEncryptedCredential` on the original `stream.path`). Since the in-memory `stream.path` is always the decrypted string after loading, the method must re-encrypt all stream paths using `encryptCredential()` before writing. This ensures credentials are always stored encrypted in `cameras.json`.
    3. Write to `cameras.json.tmp` first (temp file), then `fsp.rename()` to `cameras.json` (atomic on POSIX).
    4. Wrap in try/catch — log errors with `logger.error` but do NOT throw (persistence failure should not crash the operation).

    For the cameras.json path: `rtspManager.ts` already uses `__dirname` (line 14). The config loader resolves cameras.json as `path.join(__dirname, '../../cameras.json')` where `__dirname` is `server/src/config/`. For rtspManager.ts, `__dirname` is `server/src/streams/`, so the cameras.json path is `path.join(__dirname, '../../cameras.json')` — same resolution to `server/cameras.json`. Define this as a module-level constant `CAMERAS_CONFIG_PATH`.

    Add the required imports at the top: `import { encryptCredential } from '../services/credentialEncryption.js'` (fsp and path are already imported).

    After adding `persistCameras()`, add a `persistCameras()` call (fire-and-forget with `.catch(err => logger.error(...))`) at the END of each existing mutation method: `addCamera()` (after `this.cameras.set`), `updateCamera()` (after `Object.assign`), `removeCamera()` (after `this.cameras.delete`), and `toggleNightMode()` (at the end of the method body). These are async calls on an async method — the caller doesn't await them, so use fire-and-forget with error logging.
  </action>
