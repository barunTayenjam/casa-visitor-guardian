    **File: frontend/src/hooks/useViewportStream.ts**

    **1. StreamSlotManager class** — Export a class `StreamSlotManager` that manages max concurrent streams:
    - Constructor takes `maxSlots: number` (default 4)
    - Private `activeStreams: Set<string>` tracking which camera IDs are streaming
    - Private `pendingQueue: Array<{ cameraId: string; resolve: () => void }>` for waiting cameras
    - `async acquire(cameraId: string): Promise<void>` — if camera already active, return immediately; if slots available, add to active set; otherwise push to pending queue and return a Promise that resolves when a slot opens
    - `release(cameraId: string): void` — remove from active set; if pending queue has entries and slots available, shift the first entry, add it to active set, and resolve its promise
    - `getActiveCount(): number` — returns activeStreams.size
    - `isStreaming(cameraId: string): boolean` — returns whether cameraId is in activeStreams

    **2. useViewportStream hook** — Export a function `useViewportStream`:
    - Parameters: `elementRef: React.RefObject<HTMLElement | null>`, optional `config: { debounceMs?: number; rootMargin?: string; threshold?: number }` with defaults `{ debounceMs: 300, rootMargin: '100px', threshold: 0.1 }`
    - Returns `{ isVisible: boolean }`
    - Internal state: `const [isVisible, setIsVisible] = useState(false)`
    - Internal ref: `debounceTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null)`
    - useEffect: create IntersectionObserver on `elementRef.current` with provided config. On intersection change: clear existing timer, set new timer that calls `setIsVisible(entry.isIntersecting)` after `config.debounceMs`. Return cleanup: `observer.disconnect()` + `clearTimeout(debounceTimerRef.current)` if set
    - Dependencies: `[elementRef, config.debounceMs, config.rootMargin, config.threshold]`

    Follow the hook conventions from use-toast.ts: named export, TypeScript strict (no `any`), explicit interface types.
