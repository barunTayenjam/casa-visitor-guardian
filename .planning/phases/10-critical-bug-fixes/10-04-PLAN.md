<objective>
Fix SQL injection vulnerability in batch processing (FIX-007). Two methods use string interpolation to inject the `days` parameter directly into SQL queries: `INTERVAL '${days} days'` and `INTERVAL '${daysToKeep} days'`. Replace with parameterized queries using PostgreSQL's `INTERVAL '1 day' * $1` pattern.

Purpose: SQL injection is a critical security vulnerability — even though the current callers pass numeric values, the function signatures accept untyped parameters that could be exploited.
Output: Both SQL queries use parameterized `$1` placeholders instead of string interpolation.
</objective>
