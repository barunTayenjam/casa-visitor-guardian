# Plan 6.1: Resource Audit & Analysis

Document current resource usage and configurations to establish optimization baseline.

## Overview

Before implementing optimizations, we must understand the current resource footprint of each component. This audit will provide baseline metrics and identify the highest-impact optimization opportunities.

## Tasks

### 1. FFmpeg Process Audit
- [ ] Document current FFmpeg command-line arguments per camera
- [ ] Measure CPU usage of each FFmpeg process (perffmpeg, htop)
- [ ] Measure memory consumption of FFmpeg processes
- [ ] Identify FFmpeg processes in `server/src/streams/`
- [ ] Review RTSP URL configurations in `cameras.json`
- [ ] Check current resolution, FPS, and codec settings

**Expected Findings:**
```
Per camera (HD):
  - CPU: ~0.85 cores
  - Memory: ~110 MB
```

### 2. Docker Resource Configuration Review
- [ ] Review current docker-compose.yml limits
- [ ] Check if any memory/CPU limits are set
- [ ] Document default Docker resource allocations
- [ ] Review container restart policies
- [ ] Check log rotation configuration

**Expected Findings:**
```
Current State: No explicit limits set
  - Docker defaults may allow unlimited resource usage
  - Risk of system swap thrashing under load
```

### 3. OpenCV Detection Analysis
- [ ] Review detection parameters in `optimizedMotionDetection.ts`
- [ ] Measure detection processing time per frame
- [ ] Check detection interval configuration
- [ ] Review MOG2 background subtractor settings
- [ ] Analyze YOLO inference timing

**Expected Findings:**
```
Current Detection Pipeline:
  - Motion detection: ~5ms per frame
  - Object detection: ~50ms per detection
  - Face detection: ~30ms per face
  - Total: ~85ms per full detection cycle
```

### 4. Backend Resource Audit
- [ ] Review Node.js memory configuration
- [ ] Check Socket.io connection limits
- [ ] Measure baseline Express server memory usage
- [ ] Review Redis configuration and memory usage
- [ ] Check PostgreSQL buffer sizes

**Expected Findings:**
```
Baseline Memory:
  - Node.js: ~150 MB
  - Redis: ~30 MB
  - PostgreSQL: ~200 MB
  - Total Services: ~380 MB
```

### 5. Current Baseline Documentation
- [ ] Create resource_baseline.md with all measurements
- [ ] Document test methodology
- [ ] Capture metrics under normal and peak load
- [ ] Note any resource spikes or anomalies

**Expected Output:**
```
Baseline (2 cameras, 1 viewer):
  - CPU: ~1.5 cores (75%)
  - Memory: ~850 MB
  - Network: ~2 Mbps

Peak (2 cameras, 2 viewers + detection):
  - CPU: ~1.8 cores (90%)
  - Memory: ~1.1 GB
  - Network: ~4 Mbps
```
