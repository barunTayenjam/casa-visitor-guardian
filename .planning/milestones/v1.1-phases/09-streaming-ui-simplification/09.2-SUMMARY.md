# Plan 9.2: Simplify CameraStream to Clean Feed Summary

**CameraStream integrated with StreamPanel drawer via tap-to-toggle gesture, connection status reduced to minimal color-coded dot with conditional full overlay**

## Accomplishments
- StreamPanel drawer integrated into CameraStream with all metrics and motion state wired through
- Tap-to-toggle panel: pointer events distinguish taps from swipes (10px threshold) so swipe is reserved for camera switching
- Connection status simplified: green dot (connected), pulsing yellow (connecting), red dot (error); full overlay only for initial connection (5s auto-dismiss) or persistent errors

## Files Created/Modified
- `frontend/src/components/dashboard/CameraStream.tsx` - Added StreamPanel integration, tap-to-toggle gesture handling, simplified connection status display with conditional full overlay

## Decisions Made
- Used pointer events (onPointerDown/Move/Up) instead of click for panel toggle — allows distinguishing taps from swipes before the panel opens
- Full connection overlay auto-dismisses after 5 seconds during connecting/reconnecting states — keeps the UI clean during slow connections while still providing initial feedback
- Error state keeps full overlay but also shows brief error text under the camera name pill when overlay is dismissed by user
- StreamPanel receives `imgRef` for screenshot functionality from within the panel
