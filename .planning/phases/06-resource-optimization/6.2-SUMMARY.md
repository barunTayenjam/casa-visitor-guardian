# Plan 6.2 Summary: FFmpeg Optimization

**Executed:** 2026-03-19
**Status:** Complete

## Changes Made

### 1. Low-Resource Configuration (server/src/config/index.ts)

Added streaming configuration options:
- `LOW_RESOURCE_MODE`: Enable low-resource optimizations
- `FFMPEG_THREADS`: Number of threads per FFmpeg (default: 2)
- `DEFAULT_FPS`: Frame rate (default: 2 fps)
- `DEFAULT_RESOLUTION`: Resolution (default: 640x360)
- `STREAM_INACTIVITY_TIMEOUT`: Auto-stop timeout (default: 5 minutes)

### 2. FFmpeg Command Optimization (server/src/streams/rtspManager.ts)

Updated FFmpeg arguments to use configurable settings:
- Reduced threads: `4` → `{config.threads}` (default 2)
- Reduced FPS: `4` → `{config.defaultFps}` (default 2)
- Reduced resolution: `1920x1080` → `{config.defaultResolution}` (default 640x360)
- Lower quality: `q:v 5` → `q:v 8` for low-resource mode

### 3. Stream Lifecycle Optimization (server/src/streams/rtspManager.ts)

Added inactivity timeout:
- FFmpeg now stays running for 5 minutes after last viewer disconnects
- Prevents rapid start/stop cycles during brief reconnections
- Timer is cancelled if viewer reconnects before timeout
