<objective>
Eliminate the double frame emission bug (DOC-01) and add an integration test that verifies the Python WebSocket → Node.js → Socket.io frame relay path (DOC-08).

**Purpose:** Fix Bug 1 — every live frame is currently emitted twice to each connected frontend, doubling bandwidth and CPU. The redundant handler in index.ts was left behind when Phase 5 (plan 05-03) moved frame relay into rtspManager.wirePythonWsFrames(). The two handlers also emit different payload shapes — the index.ts handler omits the `role` field — creating a silent contract mismatch for consumers that read `role`.

**Output:** Clean, single-emit frame relay path + automated regression test preventing double-emit from recurring.
</objective>
