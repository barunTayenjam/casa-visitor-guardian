  <action>
    Fix SQL syntax bug in DetectionCache.get() method (around line 140-141):
    - The current code uses: `INTERVAL '%s seconds'` with string formatting inside a SQL string literal
    - This is incorrect because %s inside a string literal is NOT interpolated by psycopg2
    - Fix: Use `INTERVAL %s || ' seconds'` or better, use Python to format the interval: `f"INTERVAL '{ttl} seconds'"` and pass it as raw SQL (not as a parameter), OR use `NOW() - INTERVAL %s` with the value as `f'{ttl} seconds'` passed as a parameter
    - The correct approach for psycopg2: `cursor.execute("SELECT ... WHERE updated_at > NOW() - INTERVAL %s", (f'{ttl} seconds',))` — pass the full interval string as a parameter WITHOUT quotes in the SQL

    Re-enable cache cleanup thread (around line 224-226):
    - Find the commented-out cleanup thread: `# threading.Thread(target=self._cleanup_old_cache, daemon=True).start()`
    - Uncomment it to enable periodic cleanup
    - Verify the `_cleanup_old_cache` method exists and uses correct SQL (fix its INTERVAL syntax too if it has the same bug)

    The _cleanup_old_cache method likely has the same INTERVAL bug — fix it using the same pattern: pass the interval as a parameter like `(f'{cleanup_interval} seconds',)` instead of using string interpolation inside the SQL literal.
  </action>
