<interfaces>
rtspManager.ts currently handles:
- FFmpeg subprocess spawning for legacy RTSP ingestion
- Frame boundary parsing (MJPEG from FFmpeg)
- Viewer tracking and adaptive FPS
- Stream health monitoring
- Socket.io frame relay
- Reconnection logic

After simplification, it should only handle:
- Socket.io frame relay (receiving pre-encoded JPEG from Python)
- Viewer tracking (for adaptive FPS hints to Python)
- Stream health monitoring (liveness checks)

The FFmpeg subprocess spawning and frame parsing moves to the Python side
(already done in Phase 1).
</interfaces>
</context>
