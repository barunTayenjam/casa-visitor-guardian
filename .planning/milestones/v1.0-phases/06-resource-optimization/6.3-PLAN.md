# Plan 6.3: Streaming Strategy

Implement intelligent stream management to allocate resources only when needed.

## Overview

Continuous streaming wastes resources when no users are viewing. This plan implements stream-on-demand with automatic cleanup to reduce baseline resource usage.

## Tasks

### 1. Stream-on-Demand Implementation
- [ ] Track active viewers per camera
- [ ] Start FFmpeg only when first viewer connects
- [ ] Keep FFmpeg running while viewers exist
- [ ] Stop FFmpeg when last viewer disconnects

**State Machine:**
```
IDLE → (viewer connects) → STARTING → ACTIVE
ACTIVE → (last viewer disconnects + timeout) → STOPPING → IDLE
```

**Implementation:**
```typescript
interface StreamState {
  cameraId: string;
  status: 'idle' | 'starting' | 'active' | 'stopping';
  viewerCount: number;
  ffmpegPid: number | null;
  lastActivity: Date;
  inactivityTimer: NodeJS.Timeout | null;
}

async function handleViewerConnect(cameraId: string) {
  const state = getStreamState(cameraId);
  
  if (state.status === 'idle') {
    state.status = 'starting';
    await startFFmpeg(cameraId);
    state.status = 'active';
  }
  
  state.viewerCount++;
  resetInactivityTimer(state);
}

function handleViewerDisconnect(cameraId: string) {
  const state = getStreamState(cameraId);
  state.viewerCount--;
  
  if (state.viewerCount <= 0) {
    startInactivityTimer(state);
  }
}
```

**Adaptive Logic:**
```typescript
async function adjustQuality() {
  const cpuUsage = await getCpuUsage();
  const memoryUsage = await getMemoryUsage();
  
  if (cpuUsage > 70 || memoryUsage > 1400) {
    await downgradeQuality();
  } else if (cpuUsage < 30 && memoryUsage < 1000) {
    await upgradeQuality();
  }
}
```

## Resource Savings

| Scenario | Without On-Demand | With On-Demand |
|----------|-------------------|----------------|
| No viewers | ~1.2 cores, 300MB | ~0.1 cores, 50MB |
| 1 viewer | ~1.5 cores, 400MB | ~1.5 cores, 400MB |
| Peak (2 viewers) | ~1.8 cores, 500MB | ~1.8 cores, 500MB |

**Daily Savings (8 hours active, 16 hours idle):**
- CPU: ~12 core-hours saved
- Memory: ~4 GB-hours saved
