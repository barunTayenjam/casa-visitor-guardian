<objective>
Split the 3,208-line monolithic ApiService.ts into domain-specific service modules with a shared base client for HTTP configuration and error handling.

Purpose: The current monolithic API client is hard to navigate, slow to modify, and any import pulls in the entire service. Domain-specific modules improve discoverability, enable tree-shaking, and make it easier to add tests per domain.
Output: 8 new service files in services/api/ directory, reduced ApiService.ts (re-export only or removed), updated imports in consumers.
</objective>
