# Quick Task 260323-s9k: Fix streaming architecture: reduce 3 connections per camera to 1, enable mobile device access

**Created:** 2026-03-23
**Status:** Ready for execution

## Overview

Fix critical streaming architecture issues causing DDOS-like behavior (3 connections per camera) and mobile access failures. Implement deduplication at multiple layers to ensure single connection per camera.

## Tasks

### Task 1: Frontend Request Deduplication

**Files:**
- `frontend/src/services/SocketService.ts`
- `frontend/src/contexts/CameraContext.tsx`
- `frontend/src/components/live/AdaptiveCameraGrid.tsx`

**Action:**
Implement three-layer deduplication to prevent duplicate stream requests:

1. **SocketService layer** (`SocketService.ts`):
   - Add `private requestedStreams: Set<string>` property
   - Modify `requestStream()` to check `requestedStreams` set before emitting
   - Add streamKey cleanup in `stopStream()`

2. **CameraContext layer** (`CameraContext.tsx`):
   - Add `streamingCameras` state: `useState<Set<string>>(new Set())`
   - Modify `startCameraStream()` to check if camera already streaming
   - Update `streamingCameras` set on start/stop

3. **Component lifecycle** (`AdaptiveCameraGrid.tsx`):
   - Add unique keys to CameraStream components: `key={`focused-${camera.id}`}` and `key={`grid-${camera.id}`}`
   - Ensures focused view unmounts grid components (prevents 3rd connection)

**Verify:**
- Check browser DevTools Network tab
- Request stream for one camera
- Confirm only ONE `requestStream` message sent
- Switch to focused view
- Confirm no additional requests for same camera

**Done:**
All three frontend layers implement deduplication checks, preventing multiple stream requests for same camera.

### Task 2: Mobile Socket.io Configuration

**Files:**
- `frontend/src/services/SocketService.ts`
- `frontend/src/components/dashboard/CameraStream.tsx`

**Action:**
Fix mobile-incompatible Socket.io configuration:

1. **SocketService.ts** (connection options):
   - Detect mobile: `const isMobile = /iPhone|iPad|iPod|Android/i.test(navigator.userAgent);`
   - Change `forceNew: true` → `forceNew: false` (reuse connection)
   - Change `rememberUpgrade: false` → `rememberUpgrade: true`
   - Set transports: `transports: isMobile ? ['polling', 'websocket'] : ['websocket', 'polling']` (mobile-first)
   - Set timeout: `timeout: isMobile ? 30000 : 20000` (longer for mobile)

2. **CameraStream.tsx** (mobile FPS optimization):
   - Detect mobile in frame handler
   - Set `FRAME_UPDATE_INTERVAL = isMobile ? 500 : 66` (2 FPS mobile, 15 FPS desktop)
   - Throttle frame updates based on interval

**Verify:**
- Open mobile browser (iOS Simulator or Android)
- Check Network tab: Connection uses polling first
- Verify streams load successfully
- Check console: No "Socket not connected" warnings
- Confirm lower FPS on mobile (2 FPS vs 15 FPS desktop)

**Done:**
Mobile devices successfully connect using polling transport, upgrade to WebSocket, and receive optimized frame rate.
